2)what is blocking vs non blocking?

In blocking I/O, when an I/O operation is requested, the application waits for the operation to complete before moving on to the next task. 
During this waiting period, the entire application is blocked, and it cannot execute other tasks or handle new requests until the I/O operation is finished.
In non-blocking I/O, when an I/O operation is requested, the application continues to execute other tasks or handle new requests without waiting for the I/O operation to complete.

6)what is the purpose of module.exports?

When you create a local module in Node.js (a module that you define within your application), any variables, functions, 
or objects you declare within that module's file are by default private to that module. In other words, 
they are not accessible from other parts of your application unless you explicitly export them using module.exports.

7)Differnece between default export and named export?

The main difference between default export and named export lies in how you export and import modules in JavaScript.

Default Export:
1.With default export, you can only have one default export per module.
2.When you use the default export, you do not need to use curly braces {} while importing the module.
3.The default export is used when you want to export a single function, class, or object as the primary value of the module.
4.You can give any name to the default export when importing it, making it more flexible.

Named Export:
.With named export, you can export multiple functions, classes, or objects from a single module.
2.When using named export, you must use curly braces {} with the appropriate names while importing the module.
3.Named exports are used when you want to export multiple values from a module.
